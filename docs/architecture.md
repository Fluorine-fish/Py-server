### 串口通信协议

#### 1. 基础帧格式

所有通信帧均采用32字节定长格式，包含：
- 帧头 (1字节): 's'
- 消息类型 (1字节)
- 数据负载 (29字节)
- 帧尾 (1字节): 'e'

#### 2. 消息类型定义
- 0xA0: 控制指令 (上位机->下位机)
- 0xB0: 状态反馈 (下位机->上位机)
- 0xA1: 提醒指令 (上位机->下位机)
- 0xA4: 照明控制指令 (上位机->下位机)

#### 3. 提醒类型字段 (帧字节11)
- 0x00: 无提醒
- 0x01: 久坐提醒
- 0x02: 坐姿提醒
- 0x03: 语音交互

#### 4. 照明控制帧格式

使用帧中预留的字节区域实现照明控制功能：
- 字节12-15: 亮度值 (浮点数, 0.0-100.0)
- 字节16-19: 色温值 (浮点数, 2700.0-6500.0)
- 字节20: 照明模式
  - 0x00: 手动模式
  - 0x01: 自动模式
  - 0x02: 阅读模式
  - 0x03: 休息模式

## 数据流

1. 数据采集流程
```
摄像头 -> 视频流模块 -> 图像处理 -> 分析模块 -> 数据库
```

2. 提醒流程
```
检测模块 -> 决策引擎 -> 串口通信 -> 机械臂执行
```

3. 控制流程
```
Web界面 -> 后端服务 -> 串口模块 -> 机械臂/照明系统
```

4. 数据展示流程
```
数据库 -> Web API -> 前端展示 -> 用户界面
```

#### 照明控制流程
1. 用户通过Web界面调整参数
2. 前端发送AJAX请求到/api/light_control
3. 后端验证参数并构建控制帧
4. 通过串口发送到下位机
5. 接收下位机状态反馈并更新界面
